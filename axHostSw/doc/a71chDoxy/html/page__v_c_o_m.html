VCOM

This section explains, how a VCOM Application running on Kinetis board, enables a test/demo application running on PC to establish a communication session with a Secure Element.

This is very similar to RJCT Server, but instead of running on a Linux based host like iMX this runs on a FreeRTOS/NoOS based host like FRDM-K64F or FRDM-K82F.

PC Host and VCOM

Installing VCOM Drivers on Windows 7 PC

If the VCOM drivers are not installed on your PC, you could install it by choosing one of the below methods

  1. During A71CH Host Sw Setup by selecting VCOM Driver in the checkbox as shown in the image below.
    Install VCOM during Sw Setup
  2. By running the installation script from commandline with administrator privileges.
    1. Open cmd prompt with administrator privileges.
    2. run install_vcom.bat as shown in the screenshot below.
      Install VCOM later on

Steps to use VCOM for running the examples

  1. Connect FRDM-K64F with, A71CH attached to it on, the USB Interface (Not the OpenSDA USB).
  2. FRDM-K64F should have the VCOM firmware. If not, Import vcomA71CH project into MCUXpresso. Build and download the VCOM firmware. For details on MCUxpresso Build and Debug, refer Setup Kinetis/MCUXpresso
  3. Open device manager, Under Ports (COM & LPT), look for the connected device.
    Virtual Com Port COM-Number
  4. specify this COMport for connecting from PC. For e.g. COM17
