DTLS server/client

This is a DTLS Client & server demonstration program. dtls_server_sa.c and dtls_client_sa.c are examples from mbed TLS adapted to accept certificate path from commanline. ax_dtls_client.c is an extended version of dtls_client_sa.c that internally uses A71CH.

The Server and client program connects to each other and establish a DTLS Session.

The stand alone programs (dtls_server_sa.c and dtls_client_sa.c) use command line arguments to get the KEYs and certificates used to establish the session.

ax_dtls_client.c uses key pair at index 0 as private key, public key at index 0 for root CA of the counterpart server, and client certificate from GP Storage stored at address 0.

Users are encouraged to compare ax_dtls_client.c and dtls_client_sa.c to identify how the keys are associated.

For details about building and debugging the visual studio projects which create the above solutions please refer to PC build Windows.

Batch files are included to help in specifying the command line options for executing the above examples. parameter passed to the executable can be viewed in the .cmd files generated post execution of the batch files.

The below steps are to be used to run the applications using the batch scripts

Steps for stand-alone DTLS Client Server example

  1. DTLS server has to be started first by executing start_standalone_DTLS_server.bat
  2. DTLS client has to be started by executing start_standalone_DTLS_client.bat

Steps for A71CH DTLS Client Server example

  1. VCOM firmware has to be loaded and running on FRDM-K64F connected to A71CH. VCOM
  2. Keys have to preprovisioned into A71CH using the script tlsPrepareClient.sh
  3. DTLS server has to be started first by executing start_standalone_DTLS_server.bat
  4. DTLS client has to be started by executing start_a71ch_DTLS_client.bat followed by the COM port number
