mbed TLS Examples

These examples demonstrate integration of mbed TLS with A71CH. They are executed from PC and use VCOM interface to connect mbed TLS to A71CH.

These examples are split into two parts.

  1. Stand Alone Examples: These examples are stand alone examples and do not use A71CH at all. These examples depends on command line arguments to get the necessary KEYS/Certificates for the demonstration.

    These examples have _sa_ in their file name.

  2. A71CH Examples: These examples depend on A71CH for cryptographic operations. These examples do not use KEYS/Certificates from any file system, but rather use Keys at specific indexes in the A71CH or Certificates in the GP Storage of the A71CH.

    These examples have _ax_ in their file name.

Having both _sa_ and _ax_ examples would help user understand the changes needed for integration of A71CH's in context of mbed TLS.

Prerequisites

Following steps are needed to run these examples.

1) Generating keys and certificates

Default installation is not provided with any keys/certificates. But user must generate initial set of keys/certificates. <install_dir>\hostLib\mbedtls\scripts\RunOnce_CreateCertificates.bat has to be run once to create the certificates. Examples use these certificates. Once the scripts is executed, the keys and certificates will be at the folder <install_dir>\hostLib\mbedTLS\ecc

2) Setting up A71CH

  • Connect FRDM-K64F/FRDM-K82F board to PC via a the USB-VCOM interface refer to VCOM .
  • Load the keys and certificate into A71CH using the script <install_dir>hostLib\mbedTLS\scripts\ResetAndUpdateA71CH.bat.
