U16 hcNormalizeAsnSignatureEcc ( U8 *  asnSignature,
U16 *  asnSignatureLen 
)

The purpose of this function is to turn the proprietary ECDSA signature format - that may be returned by the A7x - into a normalized ASN.1 format.

The A7x will package the 'r' and 's' integer components of an ECDSA signature either as octet arrays of ASN_Pxxx_BYTE_COUNT or (ASN_Pxxx_BYTE_COUNT + 1) with the additional octet added to ensure the integer value is a positive number.

ASN.1 requires an integer value is always encoded in the smallest possible number of octets, a requirement not fulfilled by the A7x.

Note: The implementation of this function only deals with signatures associated with key lengths of 192, 224 and 256 bits
