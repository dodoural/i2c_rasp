ECDSA verify

This application demonstrates the ECDSA functionality. It takes one pre-signed certificate and a public key as input parameter. The certificate contains the digital signature which is generated using private key of the root CA. The application verifies the digital signature by generating a hash from certificate data and signing it through the public key which is contained in Secure module or host.

ECDSA verify

Stand Alone (SA) based testing

  • Pre-requisite
  • Certificates and PEM file having Public key . Refer to 1) Generating keys and certificates
  • Visual studio project "mbedTLS_sa_ecdsa_verify" and executable "mbedTLS_sa_ecdsa_verify.exe" built. Refer to PC build Windows
  • steps

mbedTLS_sa_ecdsa_verify.exe cerfilename=<CERTIFICATE File Name> pemfilename=<PEM file name> <certificate file name> is the certificate which contains the digital signature <PEM file name> is the public key in PEM file format

AX based testing

In AX based testing for digital signature verification, the public key required for verification is obtained from the secure module

  • Pre-requisite
  • Certificates and PEM file having Public key . Refer to 1) Generating keys and certificates
  • A71 CH hardware with secure element module having keys and certificate. Refer to 2) Setting up A71CH
  • Visual studio project "mbedTLS_ax_ecdsa_verify" and executable "mbedTLS_ax_ecdsa_verify.exe" built. Refer to PC build Windows
  • steps

mbedTLS_ax_ecdsa_verify.exe cerfilename=<CERTIFICATE File Name> connect=<comportname> <certificate file name> is the certificate which contains the digital signature <comportname> contains the comport through which A71 shall be accessed

Generate Certificate and Sign

This application generates X509 certificate and sign the certificate using the private key available in the secure module or host. User of the application is expected to provide minimum information such as issuers key, subject key, key usage and certificate type as input paramter. If the application is executed for AX model, the issuers key is retrieved from the secure module and used for generation of the certificate else the issuers key is provided as input parameter in case of stand alone model

Stand Alone (SA) based testing

  • Pre-requisite
  • issuer key and subject key.
  • Issuer key shall be a private key and subject key shall be a public key. Refer to 1) Generating keys and certificates
  • Visual studio project "mbedTLS_sa_Cert_gen_sign" and executable "mbedTLS_sa_Cert_gen_sign.exe" built. Refer to PC build Windows
  • steps

mbedTLS_sa_Cert_gen_sign.exe subject_key=<Subject_key> subject_name=<subject Name> issuer_key=<Issuer Key> issuer_name=<Issuer Name>key_usage=<Key usage> output_file=<output file name>

where Subject_key is the subject public key_agreement Subject Name is the name of the suject requesting certificate issuer key is the private key of the entity which is going to issue certificate issuer name is the name of the certificate issuer key_usage is the purpose of the key which is created output file is the name of the output file

For further help, type the command "mbedTLS_sa_Cert_gen_sign.exe"

AX based testing

In AX based testing for generating certificates and signing, the issuers key is obtained from the secure module for signing.

  • Pre-requisite
  • Issuer key shall be a private key and subject key shall be a public key. Refer to 1) Generating keys and certificates
  • A71 CH hardware with secure element module having issuers private keys. Refer to 2) Setting up A71CH
  • Visual studio project "mbedTLS_ax_cert_gen_sign" and executable "mbedTLS_ax_cert_gen_sign.exe" built. Refer to PC build Windows
  • Steps

mbedTLS_ax_Cert_gen_sign.exe subject_key=<Subject_key> subject_name=<subject Name> issuer_name=<Issuer Name> key_usage=<Key usage> output_file=<output file name> connect=<Comport_number>

where Subject_key is the subject public key_agreement Subject Name is the name of the suject requesting certificate issuer name is the name of the certificate issuer key_usage is the purpose of the key which is created output file is the name of the output file connect is the paramter for the comport number of the board which is connected to the host

For further help, type the command "mbedTLS_ax_Cert_gen_sign.exe"
