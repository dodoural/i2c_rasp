6 This section illustrates step-by-step the different messages exchanged between
7 client and server to setup a TLS 1.2 connection (ECDHE-ECDSA-AES128-SHA).
8 Furthermore it illustrates the support offered by the Secure Element to the client.
9 
10 # (1) Summary
11 
12 The TLS 1.2 protocol is specified in (https://tools.ietf.org/html/rfc5246).
13 The TLS demo demonstrates setting up a mutually authenticated and
14 encrypted link between a client and a server system.
15 The keypair used to identify the client is stored in the Secure Element.
16 The keypair used to identify the server is simply available as a pem file.
17 
18 The public keys associated with the respective key pairs are contained in respectively a client and a
19 server certificate.
20 
21 The signing (CA) keypair and associated certificate is a self-signed certificate.
22 The same CA is used to sign client and server certificate.
23 
24 The standard OpenSSL tools s_server and s_client are used to set-up and demonstrate the TLS link.
25 Refer to \ref page_a71ch_opensslengine_tls for the instructions that were used to set up the link.
