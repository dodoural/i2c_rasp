1 # Setup i.MX6ULEVK {#page_imx6ul}
2 
3 - DocRevision : 0.97
4 - Date : 2018-04-04
5 
6 This section explains how to set-up a cross-compilation environment for the i.MX6UL and how
7 to connect the MCIMX6UL-EVKB board to an A71CH on an I2C mini-PCB.
8 
9 \note The MCIMX6UL-EVKB board is NXP's current evaluation board for the i.MX6UL. For the purpose of evaluating the A71CH also the previous version of the board can be used (MCIMX6UL-EVK)
10 
11 # Preparation Steps
12 
13 ## (1) Downloading and Installing Yocto for i.MX6UL (MCIMX6UL-EVK board)
14 
15 Please consult first the detailed information that can be found on (\p http://www.nxp.com/imx6ULEVK).
16 Download e.g. the L4.9.11_1.0.0_LINUX_DOCS documentation package (available on \p https://www.nxp.com/products/processors-and-microcontrollers/applications-processors/i.mx-applications-processors/i.mx-6-processors/i.mx6ultralite-evaluation-kit:MCIMX6UL-EVK?tab=Documentation_Tab under the header 'Supporting Information') and take the ‘i.MX Yocto Project User’s Guide’ as a starting point.
17 
18 Yocto must be installed on a Linux PC (consult the documentation for the Linux distributions officially supported),
19 it’s possible to use a Virtual PC environment as e.g. Virtual Box.
20 The Linux PC must have access to the internet to retrieve additional packages, tools and sources.
21 
22 Having set-up all tools and packages required by Yocto, initialize a local git repository as follows:
23 
24  > mkdir ~/projects/fsl-arm-yocto-bsp
25  > cd ~/projects/fsl-arm-yocto-bsp
26  > repo init -u git://git.freescale.com/imx/fsl-arm-yocto-bsp.git -b imx-morty -m imx-4.9.11-1.0.0_ga.xml
27  > repo sync
28 
29 ## (2) Prepare an embedded linux distribution for the MCIMX6UL-EVK board
30 
31 The bitbake target \p core-image-full-cmdline created for MCIMX6UL-EVK contains a busybox implementation of
32 the usual Unix command line tools. It assumes you will interact with the embedded system over the command line.
33 
34  > cd ~/projects/fsl-arm-yocto-bsp
35  > DISTRO=fsl-imx-fb MACHINE=imx6ulevk source fsl-setup-release.sh -b build-fb-6ul
36 
37 Now, before issuing the bitbake command, first edit the file \p build-fb-6ul/conf/local.conf so it matches the following:
38 
39  MACHINE ??= 'imx6ulevk'
40  DISTRO ?= 'fsl-imx-fb'
41  PACKAGE_CLASSES ?= "package_rpm"
42  EXTRA_IMAGE_FEATURES ?= "debug-tweaks tools-debug tools-testapps package-management"
43  USER_CLASSES ?= "buildstats image-mklibs image-prelink"
44  PATCHRESOLVE = "noop"
45  BB_DISKMON_DIRS = "\
46  STOPTASKS,${TMPDIR},1G,100K \
47  STOPTASKS,${DL_DIR},1G,100K \
48  STOPTASKS,${SSTATE_DIR},1G,100K \
49  STOPTASKS,/tmp,100M,100K \
50  ABORT,${TMPDIR},100M,1K \
51  ABORT,${DL_DIR},100M,1K \
52  ABORT,${SSTATE_DIR},100M,1K \
53  ABORT,/tmp,10M,1K"
54  PACKAGECONFIG_append_pn-qemu-native = " sdl"
55  PACKAGECONFIG_append_pn-nativesdk-qemu = " sdl"
56  CONF_VERSION = "1"
57  IMAGE_ROOTFS_EXTRA_SPACE = "320000"
58 
59  DL_DIR ?= "${BSPDIR}/downloads/"
60  ACCEPT_FSL_EULA = "1"
61 
62  IMAGE_INSTALL_append += " cmake curl git subversion"
63 
64 \note If you would rather prepare an embedded linux distribution with native development tools, use the local.conf file provided in the appendix (at the bottom of this page)
65 
66 After updating the file \p build-fb-6ul/conf/local.conf issue the following command:
67 
68  > bitbake core-image-full-cmdline
69 
70 \note The directory \p build-fb-6ul will contain downloaded sources and build artefacts.
71 
72 When the above bitbake command finished successfully an sdcard image containing bootloader, filesystem and linux kernel
73 is available under \p ~/projects/fsl-arm-yocto-bsp/build-fb-6ul/tmp/deploy/images/imx6ulevk/core-image-full-cmdline-imx6ulevk.sdcard.
74 'core-image-full-cmdline-imx6ulevk.sdcard' is a symbolic link to the actual file name - which has a timestamp as part
75 of the filename e.g. 'core-image-full-cmdline-imx6ulevk-20170825222257.rootfs.sdcard'.
76 Copy this sdcard image to a microSD card either with the method described in the
77 ‘i.MX Yocto Project User’s Guide’ or on a Windows PC with e.g. the Win32DiskImager tool.
78 
79 \note The embedded linux system prepared has a user 'root' that does not require a password.
80 
81 Now one needs to apply (only once) a workaround for a problem described in the release notes of Yocto-Linux imx-4.9.11-1.0.0 (Cf. Release Notes: Table 12. Common known issues and workarounds for i.MX 6 and i.MX 7 - Network connection over Ethernet is not established automatically).
82 Hook up a terminal to the IMX6UL-EVK board over the USB-Serial link, log on and issue the following commands:
83 
84  systemctl unmask connman
85  systemctl enable connman
86  systemctl start connman
87 
88 \note Check the appendix at the bottom of this page for an alternative image that includes a native compiler and SDK.
89 
90 ## (3) Create and install SDK and Root file system on Host
91 
92  > cd ~/projects/fsl-arm-yocto-bsp
93  > MACHINE=imx6ulevk source setup-environment build-fb-6ul
94  > bitbake -c populate_sdk core-image-full-cmdline
95 
96 To install the SDK in the default location /opt/fsl-imx-fb/4.9.11-1.0.0
97 
98  > cd ~/projects/fsl-arm-yocto-bsp/build-fb-6ul/tmp/deploy/sdk/
99  > ./fsl-imx-fb-glibc-x86_64-core-image-full-cmdline-cortexa7hf-neon-toolchain-4.9.11-1.0.0.sh
100 
101 You have now installed a set of cross-compilation tools for the MCIMX6UL-EVK board on the Linux Host PC.
102 
103 ## (4) Unpacking the A71CH SW package
104 
105  > cd
106  > ls
107  ... projects ...
108 
109 Create a directory 'axHostSw' in the 'projects' directory
110 
111  > cd projects
112  > mkdir axHostSw
113  > cd axHostSw
114 
115 Retrieve the package A71CH_HostSolution_1.3_src.tgz and unpack it in the axHostSw directory
116 
117  > cd axHostSw
118  > tar xzvf A71CH_HostSolution_1.3_src.tgz
119  > ls
120  A71CH_HostSolution_1.3_src.tgz doc hostLib linux win32
121 
122 \note You may be installing a different/later version of the A71CH SW package
123 
124 ## (5) Cross-compile
125 
126 Start by setting up the appropriate environment variables for the cross compiler.
127 
128  > cd ~/projects/axHostSw/linux
129  > source /opt/fsl-imx-fb/4.9.11-1.0.0/environment-setup-cortexa7hf-neon-poky-linux-gnueabi
130 
131 Build the individual targets.
132 
133  > make -f Makefile_A71CH default app=A71CH conn=i2c platf=imx
134 
135 The following executable is created:
136 
137  A71CH_i2c_imx
138 
139 ## (6) Transfer cross-compiled application to embedded system
140 
141 As we created the application \p A71CH_i2c_imx on the host PC, it still needs to be transferred to
142 the filesystem of the embedded platform.
143 One can transfer the file via scp, ftp or use a USB stick.
144 
145 The following transcript illustrates using scp to transfer the file from the Development PC to the
146 embedded platform
147 
148  > scp A71CH_i2c_imx root@192.168.1.65:/home/root
149 
150 The following transcript illustrates using a USB stick (containing the exe)
151 to transfer the file.
152 
153  imx> cd /home/root
154  imx> mount /dev/sda1 usb # usb is mount point (directory) available in /home/root
155  imx> cp usb/A71CH_i2c_imx .
156  ..
157  imx> umount usb
158 
159 ## (7) Try out the A71CH_i2c_imx Application
160 
161 This app is built on top of the Host API.
162 
163 Ensure the A71CH-mini-PCB is connected as follows to the MCIMX6UL-EVK boards.
164 The pins required to connect the A71CH over I2C are available on connector J1704 (part of Arduino Header) of the MCIMX6UL-EVK board:
165 - Pin 7: GND
166 - Pin 8: VPERI_3V3
167 - Pin 9: I2C2_SDA
168 - Pin 10: I2C2_SCL
169 
170 The relevant pins of the connector (Pin 10 is located at the outside of the connector: the top pin inside the red circle)
171 are highlighted on the following figure: ![Connecting to MCIMX6UL-EVK board](iMX6UltraLite_Circle.png)
172 
173 The following drawing illustrates the pins to connect (using the Arduino shield pin naming conventions):
174 ![I2C pins on MCIMX6UL-EVK board's Arduino-Shield compatible connector ](ArduinoConnectorImx6ulevkb.svg)
175 
176 Go to the location in the embedded filesystem where the cross-compiled application was stored and start
177 the application:
178 
179  imx> cd 'location on embedded platform'
180  imx> ./A71CH_i2c_imx
181

208 ## (Appendix) Prepare an embedded linux distribution for the MCIMX6UL-EVK board with native compiler and SDK
209 
210 The approach is identical to the one described in section (2) of this document, please use the following local.conf file instead. Notice the two extra arguments provided to \p EXTRA_IMAGE_FEATURES: \p dev-pkgs and \p tools-sdk.
211 
212  MACHINE ??= 'imx6ulevk'
213  DISTRO ?= 'fsl-imx-fb'
214  PACKAGE_CLASSES ?= "package_rpm"
215  EXTRA_IMAGE_FEATURES ?= "debug-tweaks dev-pkgs tools-debug tools-sdk tools-testapps package-management"
216  USER_CLASSES ?= "buildstats image-mklibs image-prelink"
217  PATCHRESOLVE = "noop"
218  BB_DISKMON_DIRS = "\
219  STOPTASKS,${TMPDIR},1G,100K \
220  STOPTASKS,${DL_DIR},1G,100K \
221  STOPTASKS,${SSTATE_DIR},1G,100K \
222  STOPTASKS,/tmp,100M,100K \
223  ABORT,${TMPDIR},100M,1K \
224  ABORT,${DL_DIR},100M,1K \
225  ABORT,${SSTATE_DIR},100M,1K \
226  ABORT,/tmp,10M,1K"
227  PACKAGECONFIG_append_pn-qemu-native = " sdl"
228  PACKAGECONFIG_append_pn-nativesdk-qemu = " sdl"
229  CONF_VERSION = "1"
230  IMAGE_ROOTFS_EXTRA_SPACE = "320000"
231 
232  DL_DIR ?= "${BSPDIR}/downloads/"
233  ACCEPT_FSL_EULA = "1"
234 
235  IMAGE_INSTALL_append += " cmake curl git subversion"
