1 # i.MX6UL: Setting up OpenSSL Engine {#page_imx6ul_setup_openssl_engine}
2

6 This section explains how to compile the A71CH OpenSSL Engine and associated example programs.
7 It details how to install binaries, scripts and associated key material on the embedded platform.
8 
9 \note **NXP makes available an MCIMX6UL-EVK(B) specific SD card image that comes with native development tools and both the A71CH HostLib and OpenSSL engine pre-installed. Jump to step (5) when using this SD card.**
10 
11 \note First consult \ref page_imx6ul for instructions on setting up a cross compilation environment and
12 compiling the A71CH Host SW.
13 
14 \note In case a native compilation environment is available on the Embedded Target one could skip setting
15 up the cross compilation environment. Jump to step (4) below and unpack the full source code
16 tree in /home/root/axHostSw instead, compile the binaries as outlined in step (2) and (3) and create
17 the two symbolic links.
18 
19 \note To ensure the OpenSSL engine is used at run-time by an application be sure to define the flag \p OPENSSL_LOAD_CONF when compiling the application. Please check the makefiles provided with the examples for further details. More information on the OpenSSL engine is available in the document 'A71CH OpenSSL Engine [um4334**]'.

21 # (1) Pre-conditions
22 
23 The A71CH Host SW has been unpacked in a directory axHostSw.
24 The development PC has been prepared for cross compiling for the MCIMX6UL-EVKB board.
25 The unpacked SW tree looks like
26

45 Use cmake in cross compilation mode to create make files and applications for the embedded platform.
46 Invoke make and install the A71CH shared library (libA71CH_i2c.so.1.4.0), HLSE shared library (libHLSE_A71CH_i2c.so.1.0.0) and associated header files in a standard system location of the SDK target system root.

76 # (2) Cross compile
77 
78 To create the OpenSSL Engine shared library, execute the following command
79 
80  > cd axHostSw/linux
81  > make -f Makefile_A71CH engine app=A71CH_LINKED_ENGINE conn=i2c platf=imx
82 
83 To successfully run the OpenSSL examples the A71CH must be provisioned with appropriate credentials.
84 Provisioning is done with the A71CH configure application, which is created as follows:
85 
86  > make -f Makefile_A71CH default app=A71CH_CONFIG conn=i2c platf=imx
87 
88 After executing the above make commands the axHostSw/linux directory will contain (at least):
89 
90  > tree axHostSw/linux
91  axHostSw/linux
92  ├── a71chConfig_i2c_imx
93  ├── buildA71CH.sh
94  ├── libe2a71chi2c.so
95  ├── libe2a71chi2c.so.1.0.0
96  └── Makefile_A71CH
97 
98 
99 The A71CH Host SW also contains two example applications using the OpenSSL Engine. To compile
100 these examples go first into directory `axHostSw/hostLib/embSeEngine/a71chEx` and invoke the following commands:
101 
102  > cd axHostSw/hostLib/embSeEngine/a71chEx
103  > make -f Makefile_a71chTlsClient conn=i2c platf=imx ENGINE_HINT=ON
104  > make -f Makefile_a71chEcDhKa conn=i2c platf=imx ENGINE_HINT=ON
105 
106 After executing these commands the `axHostSw/hostLib/embSeEngine/a71chEx` directory will
107 contain the following files:
108 
109  > tree axHostSw/hostLib/embSeEngine/a71chEx
110  axHostSw/hostLib/embSeEngine/a71chEx
111  ├── a71chEcDhKa
112  ├── a71chEcDhKa.c
113  ├── a71chEcDhKa.o
114  ├── a71chTlsClient
115  ├── a71chTlsClient.c
116  ├── a71chTlsClient.o
117  ├── Makefile_a71chEcDhKa
118  └── Makefile_a71chTlsClient

120 \note To ensure the OpenSSL engine is used at run-time by an application be sure to define the flag \p OPENSSL_LOAD_CONF when compiling the application. Please check the makefiles provided with the examples for further details. More information on the OpenSSL engine is available in the document 'A71CH OpenSSL Engine [um4334**]'.

122 # (3) Packaging OpenSSL Engine related files for transfer to embedded platform.
123 
124 To be able to execute the OpenSSL Engine examples one needs to transfer
125 binaries, scripts and associated key material to the embedded platform.
126 The script `axHostSw/linux/packageA71chOpenSslEngine.sh` will collect these files and put them in a compressed
127 tar archive `A71CH_Engine.tgz`. The script \b must be invoked from the `axHostSw/linux` directory.
128 
129  > cd axHostSw/linux
130  > ./packageA71chOpenSslEngine.sh
131 
132 Now transfer `A71CH_Engine.tgz` to the embedded platform (e.g. using scp, ftp or a USB stick).
133 
134 In addition one must transfer the shared libraries created on the SDK target system root to the embedded platform.
135 Assuming default locations of the cross compiler, this implies copying the content of the directories
136 
137  /opt/fsl-imx-fb/4.9.11-1.0.0/sysroots/cortexa7hf-neon-poky-linux-gnueabi/usr/local/bin
138  /opt/fsl-imx-fb/4.9.11-1.0.0/sysroots/cortexa7hf-neon-poky-linux-gnueabi/usr/local/lib
139 
140 To the equivalent directories on the embedded platform.
141 
142  /usr/local/bin
143  /usr/local/lib

204 
205 Create a symbolic link to the OpenSSL config file that configures OpenSSL for usage with the A71CH OpenSSL Engine
206 
207  >> mkdir -p /etc/ssl # Create directory '/etc/ssl' in case it does not yet exist
208  >> ln -s /home/root/axHostSw/hostLib/embSeEngine/info/opensslA71CH_i2c.cnf /etc/ssl/opensslA71CH_i2c.cnf
209 
210 Create symbolic links to the A71CH shared library and OpenSSL engine (shared library)
211 
212  >> ln -s /home/root/axHostSw/linux/libe2a71chi2c.so.1.0.0 /usr/lib/libe2a71chi2c.so
213 
214 Ensure the location of the shared libraries libA71CH_i2c.so and libHLSE_A71CH_i2c.so is known by running the following command
215 once
216 
217  >> ldconfig /usr/local/lib
