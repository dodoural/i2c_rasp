6 This section explains how an RJCT server enables a client process (remote or local)
7 to establish a communication session with a Secure Element.
8 
9 # (1) Introduction
10 
11 The RJCT server is a standalone process that can establish a communication session
12 with a Secure Element on behalf of a client process. Only one client process (at a time)
13 can have a communication session via the RJCT server with the Secure Element.
14 
15 The RJCT server listens for incoming connection requests from client processes on port 8050.
16 
17 The following figure illustrates a client application (in the example the Configure Tool)
18 connecting to the RJCT server,

22 # (2) Using the RJCT server
23 
24 The RJCT server application that is bundled with the Host SW is a command line application.
25 In the following example the server is started from a shell on the embedded platform
26 
27  root@imx6ulevk:~/projects/axHostSw/linux$ ./rjct_a71_i2c_native
28  RemoteJCShell Terminal Server (supporting SCI2C) Rev 0.92
29  A71 (I2C_CLK_MAX = 400 kHz) - Effective Master Clock depends on Host Platform.
30  ******************************************************************************
31  Establish a connection via JCShell:
32  /term Remote|<ip-address>:<port>
33  e.g.
34  /term Remote|192.168.1.27:8050
35 
36  Server: waiting for connections...
37 
38 For a client to connect to the server it needs to know the IP address of the server.
39 In the following example the Configure Tool - built with the smComSocket_linux communication
40 layer - runs on a Linux PC and connects to the RJCT server running on the embedded platform
41 (with IP address 192.168.1.100).
