OpenSSL Engine: Miscelaneous example
  • DocRevision : 0.91
  • Date : 2017-09-08

This section explains how to execute a set of examples illustrating the usage of the A71CH OpenSSL Engine. (Consult i.MX6UL: Setting up OpenSSL Engine for information on how to prepare the embedded platform.)

(1) Summary

The set of examples covered in this section illustrate how to use the standard OpenSSL tools in combination with the OpenSSL engine to

  • Fetch random numbers
  • Create a Certificate Signing Request with the private key stored in the Secure Element
  • Sign a message with a private key stored in the Secure Element
  • Verify a message with a public key stored in the Secure Element

Furthermore the ECDH example illustrates how to activate the usage of the OpenSSL engine from a C-program built upon the OpenSSL API.

To run all these examples successfully one needs to create the appropriate credentials upfront and inject them in the Secure Element. The script covering this step assumes the Debug Interface is available on the Secure Element.

(2) Secure Element preparation

Execute the a71chPrepareEcc.sh script once before trying out any of the examples covered in this section.

Note
Executing the script will change the state of the attached A71CH Secure Element.
> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./a71chPrepareEcc.sh

(3) Fetch random numbers

> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./a71chRandDemo.sh

(4) Create a CSR

> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./a71chEccCsrDemo.sh

(5) Sign and verify messages

> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./a71chEccSignDemo.sh file  # Provide a file-path as argument to the script

(6) ECDH example

> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./a71chEcDhKa.sh
