OpenSSL Engine: TLS example
  • DocRevision : 0.92
  • Date : 2018-04-04

This section explains how to set-up a TLS demo using the A71CH OpenSSL Engine. (Consult i.MX6UL: Setting up OpenSSL Engine for information on how to prepare the embedded platform.)

(1) Summary

The TLS demo demonstrates setting up a mutually authenticated and encrypted link between a client and a server system. The keypair used to identify the client is stored in the Secure Element. The keypair used to identify the server is simply available as a pem file.

The public keys associated with the respective key pairs are contained in respectively a client and a server certificate.

The signing (CA) keypair and associated certificate is a self-signed certificate. The same CA is used to sign client and server certificate.

The TLS demo comes in two flavours:

  1. Flavour-A: The standard OpenSSL tools s_server and s_client are used to set-up and demonstrate the TLS link. The certificates used are simply stored on the file system of the host. The client uses the keypair stored inside the A71CH.
  2. Flavour-B: A TLS client program - included in source code (a71chTlsClient.c) - retrieves the client certificate from the A71CH and uses the keypair stored inside the A71CH. It establishes a TLS connection with the server process s_server.

Steps (2) to (5) are identical for the two demo flavours.

(2) Credential preparation (execute once) [Optional]

> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./tlsCreateCredentialsRunOnClientOnce.sh
Note
The Host SW package comes bundled with the required credentials. The tlsCreateCredentialsRunOnClientOnce.sh script will re-create equivalent credentials (but with new/different keypairs)

To simplify the credential creation process the script creates both client and server credentials (once) on the client platform. One must transfer the server credentials created to the server platform.

(3) Secure Element preparation (client side)

For the purpose of the demo one MUST inject the TLS client key pair into the Secure Element and create a reference pem file referring to this provisioned key pair. 

> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./tlsPrepareClient.sh

(4) Server side preparation

Note
The Host SW package comes bundled with the required server credentials.

Transfer the default server credentials or those created under (2) to the server platform.

(5) Start up the server

Note
The server can run e.g. on a PC. The server must be reacheable over the TCP/IP network for the Client.

Execute the following command on the server platform, make a note on the IP address of the server. 

> cd axHostSw/hostLib/embSeEngine/a71chDemo/scripts
> ./tlsServer.sh [ECDHE|ECDH|both|max]

(6) Establish a TLS link from the client to the server

The client process establishing the TLS connection comes in two flavours: either s_client or a program provided in source code (a71chTlsClient.c). Invoke either example through a bash shell script.

(6.1) Using s_client

Invoke the script using the IP address of the server as the first argument and ECDH or ECDHE as the second argument (ECDHE corresponding to ECDH ephemeral). 

> ./tlsSeClient.sh <server-IP-address> [ECDH|ECDHE]

(6.2) Using a71chTlsClient.c

Invoke the script using the IP address of the server as argument. 

> ./a71chTlsClient.sh <server-IP-address>
